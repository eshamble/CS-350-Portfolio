  Summarize the project and what problem it was solving.
The final project in CS-350 was a thermostat. It had three states: off, heat, and cool. One button could toggle between the states, and the other two incremented and decremented the desired temperature so that the lights could indicate whether the heating or cooling was active or not. There was a LCD display that would show the user the date, time, state, current room temperature, and deaired room temperature.

  What did you do particularly well?
The coding process was rather smooth when I got to it.

  Where could you improve?
I'm not sure how to fix this, but the buttons were a bit too sensitive, so they would sometimes increment/decrement too much. Also, I turned the project in really late because I was struggling very badly with time management this semester.

  What tools and/or resources are you adding to your support network?
When I asked classmates about the numbering system on the GPIO board, I got very helpful replies. I need to make better use of support discussion boards in my classes.

  What skills from this project will be particularly transferable to other projects and/or course work?
Making the state machine diagram involved a lot of thinking about how the project works. Creating documentation like that before starting the code is a lfie saver when it comes to understanding it.

  How did you make this project maintainable, readable, and adaptable?
Most of the project architecture was done for me, but I did use the provided variables, etc, consistently, so I didn't mess with the structure. I also kept comments on all of the changes I made (the language of these comments was from the TO-DOs).
